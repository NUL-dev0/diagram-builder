# コントリビュートガイド

DiagramBuilder へのコントリビューションを歓迎します。

## 開発環境のセットアップ

README の「開発環境のセットアップ」を参照してください。

## ブランチ戦略

- `main` — 安定版。直接プッシュ禁止。
- `feature/*` — 新機能
- `fix/*` — バグ修正
- `chore/*` — ドキュメント・設定変更

## プルリクエストの手順

1. `main` から新しいブランチを作成
2. 変更を実装・テスト
3. コミット前チェック（husky が自動実行）
   - ESLint による静的解析
   - Prettier によるフォーマット
4. プルリクエストを作成し、変更内容を説明

## コミットメッセージ

`type: 簡潔な説明` の形式で記述してください。

| type | 用途 |
|------|------|
| `feat` | 新機能 |
| `fix` | バグ修正 |
| `docs` | ドキュメント |
| `chore` | ビルド・設定変更 |
| `refactor` | リファクタリング |

## セキュリティ

- `eslint-plugin-security` によるセキュリティチェックが必須です
- API キー・機密情報をコミットしないでください
- 脆弱性を発見した場合は Issue ではなくメールで連絡してください
